## 🚀 كيفية الوصول إلى لوحة التحكم

بعد نشر الموقع، يمكنك الوصول إلى لوحة التحكم من خلال:

```
https://your-domain.com/admin/login
```

أو مباشرة:

```
https://your-domain.com/admin/dashboard
```

سيتم إعادة توجيهك تلقائياً إلى صفحة تسجيل الدخول إذا لم تكن مسجلاً دخولاً.

---

## 📝 ميزات لوحة التحكم

### 1. إضافة مقال جديد
- اضغط على زر "+ مقال جديد"
- أدخل البيانات المطلوبة:
  - **الرابط (Slug)**: معرّف فريد للمقال (مثل: ahmed-bin-said)
  - **العنوان**: عنوان المقال
  - **التصنيف**: اختر من (شخصيات، أحداث، معارك، دول، عام)
  - **التاريخ**: تاريخ المقال
  - **الوصف القصير**: ملخص قصير للمقال
  - **المحتوى**: محتوى المقال الكامل (يدعم HTML)
- اضغط "حفظ المقال"

### 2. تعديل مقال موجود
- اضغط على زر "تعديل" بجانب المقال
- عدّل البيانات المطلوبة
- اضغط "حفظ المقال"

### 3. حذف مقال
- اضغط على زر "حذف" بجانب المقال
- أكّد عملية الحذف
- سيتم حذف المقال نهائياً

### 4. تسجيل الخروج
- اضغط على زر "تسجيل الخروج" في الزاوية العلوية اليسرى

## 🔧 التكوين والأمان

### تغيير كلمة المرور

إذا أردت تغيير كلمة المرور، اتبع الخطوات التالية:

1. **توليد hash جديد** لكلمة المرور:

```bash
node -e "const bcrypt = require('bcryptjs'); console.log(bcrypt.hashSync('your_new_password', 10))"
```

2. **تحديث ملف `.env`**:

```env
ADMIN_USERNAME=asaad_968
ADMIN_PASSWORD_HASH=<paste_the_new_hash_here>
```

3. **إعادة نشر الموقع**

### ملفات الأمان

- **`src/lib/auth.server.ts`** — منطق المصادقة والتشفير (server-only)
- **`src/lib/api/auth.functions.ts`** — وظائف تسجيل الدخول والخروج
- **`src/lib/api/articles.functions.ts`** — وظائف إدارة المقالات
- **`.env`** — متغيرات البيئة (احفظها بأمان، لا تشاركها)

## 🔒 معايير الأمان المطبقة

1. **تشفير كلمات المرور** — استخدام bcryptjs مع 10 جولات
2. **معالجة server-side** — جميع العمليات تتم على الخادم فقط
3. **التحقق من الجلسات** — كل عملية تتطلب token صحيح
4. **عدم كشف البيانات الحساسة** — لا تُرسل كلمات المرور أو hashes للمتصفح
5. **متغيرات البيئة** — الأسرار محفوظة في `.env` وليست في الكود

## ⚠️ ملاحظات مهمة

1. **احفظ بيانات الدخول في مكان آمن** — لا تشاركها عبر البريد الإلكتروني
2. **غيّر كلمة المرور بانتظام** — خاصة إذا كان الموقع عام
3. **استخدم HTTPS فقط** — لا تستخدم HTTP
4. **لا تضع `.env` في Git** — أضفها إلى `.gitignore`
5. **احفظ نسخة احتياطية من المقالات** — قبل حذفها
